```text

Headers:
  Origin:  https://windsurf.com
  Referer: https://windsurf.com/account/login
  Accept:  application/json, text/plain, */*
  Content-Type: application/json
Body:
  { "email": "user@example.com", "password": "secret123" }

Response (200):
  { "token": "eyJ...", "user_id": "user-uuid-001" }
```


**流**:

1. 用户输 email + password · 点 "🔑 自动登"
2. 前端 `autoSigninWindsurf()`:

   - 校验 vmUrl 已设

3. VM 后端走 3-step · 返 `{ apiKey, apiServerUrl, sessionToken, ... }`
4. 前端自动入号库 (`D.accounts.push`) + 标 `viaSignin: true`
5. 自动设为 active · markDirty · renderMid
6. 用户即可走反代 `/v1/messages`

## 七 · 限 + 后续展

**当前限**:
- 仅支 windsurf.com 之 password login (即「devin」品牌后之 SSO)
- 不支 Google/GitHub OAuth (windsurf 官方亦走此 SSO · 但需 browser callback · 走 device-code 模式可解 · 后续印 130 立)
- 不支 2FA (主公账号若启 2FA · 需在 windsurf web 手关 · 或印 130 加 TOTP 输框)
- 不自动 refresh sessionToken (失效后用户重登 · 后续印 130 加 keepAlive)

**后续展** (印 130 候):
- A. GitHub OAuth flow (PKCE) · 代用户走 GH 登 windsurf
- B. 2FA / TOTP 支持 · 加输框 · 走第 4 步
- C. sessionToken 自动续 (印 95 之 5min keepalive 模)
- D. 多账号批量登 (`POST /admin/signin/windsurf/batch [{email,password}]`)
