## 印 130 两面 (一线到底)

### 面一 · 公网登 (OAuth Device-Flow)

用户公网登 GitHub · 一钮代 PAT · 真去中心化:

```
[gate 页] 🔑 一键登 GitHub (Device-Flow)
    ↓ 点
window.daoOAuth.start({ scope: 'repo gist workflow', onCode, onPoll, ... })
    ↓ POST https://github.com/login/device/code
        { client_id, scope: 'repo gist workflow' }
    ↓ 返
        { device_code, user_code: 'DAOX-1234',
          verification_uri: 'https://github.com/login/device',
          interval: 5, expires_in: 900 }
    ↓ UI 显 user_code · window.open(verification_uri) 新窗
    ↓ 用户在 GH 页输 8 位 code · 选授权范围 · 确认
    ↓ App 轮询 POST https://github.com/login/oauth/access_token
        { client_id, device_code, grant_type: 'urn:ietf:...:device_code' }
        ↻ 返 { error: 'authorization_pending' } → 等 interval (5s) 再轮
        ↻ 返 { error: 'slow_down' } → interval += 5
    ↓ 终
        { access_token: 'gho_...', token_type, scope }
    ↓ window.daoSync.setPat(access_token) // 复用 PAT 同槽 (帛书廿二)
    ↓ daoSync.whoami() → renderOnboarding / enterMine (PAT 同路径)
```

**关键真**:
- GH `/login/oauth/access_token` 自 2022-05-10 起支持浏览器 CORS  
  (blog: 「Supporting CORS preflight requests on github.com」)
- 故浏览器可直调 · 无需后端中转 · **真去中心化**
- OAuth access_token 与 PAT API 等价 (`Authorization: token <X>`)
- 拿到 token → `daoSync.setPat(token)` 入 `localStorage.dao_pat` → 自动复用 `oneShot` 全链路 (`fork` → `actions` → `pages` → `gist` → `pool-gist` → `dispatch` → `poll` → `probe` → `write` → `redirect`)

## 反者道之动 之真显

**正面 (常人之见)**: 给用户加 OAuth 登录是「**加**」功能 — 多写 370 行 dao_oauth.js + UI · 是为「益」。

**反面 (道之动)**:
- PAT 流之繁: 用户去 settings/tokens · 选 fine-grained · 勾 7 个 scope · 设到期日 · 命名 · 复制 · 粘到 web → 损去 5 步
- OAuth 流之朴: 用户点钮 · GH 页输 8 位 code · 确认 → 3 步
- **是为损** · 不是加

**主公诏「公网登录后端连接同步一切」之真**:
- 不是「再加一个登录入口」 (益)
- 是「**让用户的真实 GH 身份直接驱动后端**」(损 · 移走 PAT 这道墙)
- daoSync.setPat 同槽 · 圣人执一 · 万处自动受益 (无为而无不为)

**「去中心化」之真**:
- 无 broker · 无中间端 · 浏览器 ↔ github.com 直接 (CORS 通)
- 主公的 OAuth App 仅是「**颁发器**」 · token 由 GH 直返用户浏览器 · 主公永不见
- 用户的 fork + Gist + Pages · 完全用户自主 · 即便主公的 OAuth App 失效 · PAT 兜底仍通
