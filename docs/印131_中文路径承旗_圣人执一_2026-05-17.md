## 真本源症

run_all.cjs (节制 23 守门套件之父) spawn 子进程跑各 `_seal*_smoke.cjs`. Node v24 + Windows + **中文路径** (`E:\道\道生一\一生二\Windsurf万法归宗\…`) + **Junction 链** (NTFS Junction) 组合下:

```
node:fs:610
  return binding.realpath(...)
                 ^
Error: ENOENT: no such file or directory, realpath 'E:\道\道生一\…\tests\_seal128_yiqi_sanqing_smoke.cjs'
    at Object.realpathSync (node:fs:610:20)
```

**真因**:
- Node v24 之 `module._resolveFilename` 默调 `fs.realpathSync`
- `realpathSync` 对中文路径之 NTFS Junction 解析返 `ENOENT` (Win API `CreateFileW` 接受中文但 Node v24 之内层 cache 不接)
- 子进程承父之 cwd / argv 而**不承** `process.execArgv` 之 Node 旗 (`--preserve-symlinks` / `--preserve-symlinks-main`)

**实证**: 单跑 `node tests/_seal128_yiqi_sanqing_smoke.cjs` 直走 — 通. 经 `spawnSync(process.execPath, [script])` 中转 — 偶崩.

## 治本动作

`tests/run_all.cjs` 加之 (印 131 节):

```js
// 印 131 · 反者道之动 · 中文路径下子进程承旗 · 圣人执一以为天下牧
const _childExecArgv = Array.isArray(process.execArgv)
  ? [...process.execArgv]
  : [];
for (const _flag of ["--preserve-symlinks", "--preserve-symlinks-main"]) {
  if (!_childExecArgv.includes(_flag)) _childExecArgv.push(_flag);
}

for (const t of TESTS) {
  const script = path.join(__dirname, `${t}.cjs`);
  const r = spawnSync(process.execPath, [..._childExecArgv, script], {
    stdio: "inherit",
    cwd: path.join(__dirname, ".."),
  });
  // ...
}
```

**两旗合一**:
- `--preserve-symlinks` · 治 `require()` 内层之 realpath cache
- `--preserve-symlinks-main` · 治 main script 之 realpath (entry point)

**幂等**: 若父进程已带 (主公手动 `node --preserve-symlinks tests/run_all.cjs`), `process.execArgv` 已含 · 跳过加; 否则添之. **道法自然**.
